# Memory System (3-уровневая)

## Архитектура

```
┌──────────────────────────────────────────────────────┐
│   3. [[infrastructure/LightRAG|LightRAG]] (Graph RAG)   │  ← Долговременная, ассоциативная
├──────────────────────────────────────────────────────┤
│   2. Wiki / Obsidian (структурированные заметки)     │  ← Среднесрочная, связанная
├──────────────────────────────────────────────────────┤
│   1. Daily Notes (memory/YYYY-MM-DD.md)              │  ← Краткосрочная, сырая
└──────────────────────────────────────────────────────┘
```

## Уровни

### Уровень 1: Daily Notes
- Хранятся в `memory/YYYY-MM-DD.md` в workspace агента
- Сырые логи: события, решения, контекст сессий
- Ведёт каждый агент самостоятельно
- Срок жизни: несколько дней, затем дистилляция в wiki

### Уровень 2: Wiki / Obsidian
- `[[cross-links]]` — Obsidian-совместимые ссылки
- Структурированные заметки: агенты, проекты, инфраструктура
- Зеркало на GitHub (автокоммит каждые 2 минуты)
- `index.md` — корневая навигация
- Изменения отслеживаются через `.lightrag-track.json`

### Уровень 3: LightRAG (Graph RAG)
- Ассоциативная память с графом знаний (79 нод, 126 связей)
- Поиск через `query()` и `adelete_by_doc_id()`
- DeepSeek `deepseek-v4-pro` — LLM провайдер
- Разделы понимаются через `[[cross-links]]` и связи в графе
- Синхронизация: `/data/lightrag/wiki/wiki-autosync.sh` (каждый час)

## Процесс

1. **Автоинжест**: агент сам заносит важные обсуждения в wiki + LightRAG
2. **Auto-sync**: cron → git pull → удалённые файлы → LightRAG cleanup → git commit
3. **Memory search**: bge-m3 через Ollama на сервере (100.107.127.17:11434)
4. **Ingest.py**: `/data/lightrag/wiki/ingest.py` — поддерживает `--sync`, `--path`
