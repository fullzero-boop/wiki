# Маркетинговая структура: Gost 👾

## 1. Продукт

**Технический ассистент — автоматизация, парсинг, DevOps.**
Gost — engine room команды. Не внешний продукт, а внутренняя сила, которая делает возможным всё остальное.

**Роль в системе:**
- Jarvis/Sinabon/Tsilya/Paladin — создают контент
- Belfort — анализирует рынок
- Shadow — оркестрирует
- **Gost — делает так, чтобы это работало технически**

## 2. Ключевые функции

### Для контент-команды:
- Автоматизация публикаций (cron-постинг)
- Парсинг аудиторий и метрик для Hermes
- Инструменты для создания визуала (карточки Sinabon, графики Belfort)

### Для Shadow:
- Интеграция SmartHelperBot с Telegram API
- Docker-инфраструктура
- Мониторинг и логирование

### Для Hermes:
- Парсинг конкурентов (облачные парсеры, API)
- Сбор данных для аналитики
- Cap-солверы, прокси, обход блокировок

### Для себя (Gost):
- Автоматизация фарма в Albion Online
- gost-desktop (Xvfb + VNC + браузер)

## 3. Текущая задача: 30 почт

**Проблема:** ProtonMail не заводится через Chromium на сервере.
**Решение:** 
- Купить готовые почты (Gmail/Яндекс) через smm-панели
- Или завести почты на своём домене (reg.ru уже есть доступ)
- Или использовать Mail.tm API (без браузера)

**Распределение:**
- 10 почт → Shadow (для SmartHelperBot, регистраций)
- 10 почт → Jarvis (для соцсетей, площадок)
- 10 почт → Gost (для фарма, тестов)

## 4. Что продвигать через Gost

Gost не продаётся как продукт, но его работа создаёт **имидж технологичности** всей команды.

**Форматы контента:**
- **Кейсы автоматизации:** "Как я автоматизировал публикацию контента для Jarvis"
- **Технические разборы:** "Парсинг конкурентов без блокировок"
- **Open-source вклад:** выкладка полезных инструментов на GitHub

**Целевая аудитория (для кейсов):**
- Разработчики и DevOps
- Команды автоматизации
- Потенциальные партнёры (B2B)

## 5. Система метрик

- **Эффективность = задачи выполненные / задачи запрошенные**
- **Uptime инструментов**
- **Скорость = время от задачи до результата**
- **Количество сэкономленных часов команды**
- **Технический долг** (обратная метрика)

**KPI (постоянно):**
- Время реакции на задачу < 1 час
- Uptime инструментов > 99%
- Решённые задачи: 0 в очереди > 24 часов
